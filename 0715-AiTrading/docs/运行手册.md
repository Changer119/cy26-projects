# AI 模拟交易系统运行手册

## 1. 用途与当前状态

本系统在本机上对 A 股和港股做 AI 辅助的模拟交易。初始模拟资金为 100,000 元人民币。

> 安全边界：系统仅会创建本地模拟订单和模拟成交，没有券商适配器，不读取、不保存、不使用任何证券账户凭据。

当前默认自选股为：

| 代码 | 名称 | 当前模拟交易状态 |
| --- | --- | --- |
| `603005.SH` | 晶方科技 | 已配置 100 股整手，可在数据闸门通过后模拟交易 |
| `600584.SH` | 长电科技 | 已配置 100 股整手，可在数据闸门通过后模拟交易 |
| `02513.HK` | 智谱 | 仅展示；整手、港币资金和汇率尚未接入，不可交易 |
| `01810.HK` | 小米集团-W | 仅展示；整手、港币资金和汇率尚未接入，不可交易 |

目前尚未启用自动发现科创板、创业板、其他 A 股或场内基金。因此，“新增标的后飞书通知”的运行路径也尚未触发。

## 2. 运行前提

- macOS，用于本机 `launchd` 常驻。
- Python 3.12 及 `uv`。项目固定使用根目录的 `.venv`。
- Node.js 与 npm，用于构建 React 仪表盘。
- DeepSeek API Key。DeepSeek 只能返回 `BUY` / `SELL` / `HOLD`、置信度和理由，不能指定限价、止损价、目标权重或目标数量，也不能越过本地数据门和风控。
- Tiingo Token。当前实现必须用 Yahoo Finance 与 Tiingo 双源交叉验证；缺少 Token 时所有标的会安全转为 `HOLD`。请自行确认所用 Tiingo 账户的当前额度与许可条款。
- 需要飞书通知时，本机还必须已安装、登录并配置 `lark-cli`。

## 3. 首次安装与配置

所有命令均在项目根目录执行。

### 3.1 安装后端依赖

```bash
./scripts/setup.sh
```

脚本会通过 `uv sync --all-groups` 创建或更新 `.venv`。

### 3.2 安装并构建前端

```bash
./scripts/web/setup.sh
./scripts/web/build.sh
```

生产构建产物位于 `web/dist/`，后端会在 `/` 路径托管该仪表盘。`start.sh` 和常驻服务在发现产物不存在时也会自动构建。

### 3.3 配置 `.env`

推荐先在当前终端设置密钥和飞书收件人，再由安全脚本写入权限为 `0600` 的文件：

```bash
./scripts/config/save_from_env.sh
```

该脚本不回显密钥。它要求当前环境至少已有 `DEEPSEEK_API_KEY` 和 `FEISHU_TARGET_ID`；Tiingo Token 可以稍后补充，但缺失期间系统只会 `HOLD`。

也可以手工创建：

```bash
cp .env.example .env
chmod 600 .env
```

然后编辑 `.env`：

```dotenv
AI_TRADING_DATABASE_URL=sqlite:///data/ai_trading.db
AI_TRADING_LOG_LEVEL=INFO
DEEPSEEK_API_KEY=填入本机密钥
TIINGO_API_KEY=填入数据源Token
FEISHU_TARGET_TYPE=user
FEISHU_TARGET_ID=填入飞书用户ID
```

| 变量 | 是否必需 | 说明 |
| --- | --- | --- |
| `AI_TRADING_DATABASE_URL` | 否 | 默认是项目内 `data/ai_trading.db` |
| `AI_TRADING_LOG_LEVEL` | 否 | 默认 `INFO` |
| `DEEPSEEK_API_KEY` | 要生成 AI 提案时必需 | 未配置或鉴权失败时不会下模拟订单 |
| `TIINGO_API_KEY` | 当前模拟交易必需 | 缺失时不构造单源行情，全部 `HOLD` |
| `FEISHU_TARGET_TYPE` | 否 | `user` 或 `chat`，默认 `user` |
| `FEISHU_TARGET_ID` | 要发飞书时必需 | 用户 ID 或群聊 ID，与 `FEISHU_TARGET_TYPE` 匹配 |

`.env` 已被 Git 忽略。不要将密钥写入文档、命令行参数或日志。

## 4. 启动、停止与常驻

### 4.1 前台无关的本机后台启动

```bash
./scripts/start.sh
./scripts/status.sh
```

启动脚本会等待健康检查通过后返回。访问：

- 仪表盘：`http://127.0.0.1:8765/`
- OpenAPI：`http://127.0.0.1:8765/docs`
- 健康检查：`http://127.0.0.1:8765/api/health`

首次启动会创建 SQLite 结构、10 万元模拟账户和 4 个默认自选标的。使用旧版 SQLite 数据库启动时，系统会自动检查 `trade_proposals`，并为缺失的 `reason`、`strategy_version`、`target_weight_micros`、`reference_price_micros`、`stop_price_micros` 字段执行幂等增列；旧记录对应字段保持为 `NULL`。这不是通用版本化迁移框架，升级前仍应按第 8 节备份数据库。

停止手动启动的进程：

```bash
./scripts/stop.sh
```

`scripts/status.sh` 同时识别手动 PID 和 8765 端口上的健康常驻服务。

### 4.2 安装 macOS `launchd` 常驻服务

先停止手动启动的同端口进程，再安装：

```bash
./scripts/stop.sh
./scripts/service/install.sh
```

服务标识为 `com.local.ai-trading`，配置文件生成到 `~/Library/LaunchAgents/com.local.ai-trading.plist`。该服务设置了 `RunAtLoad` 和 `KeepAlive`。

卸载常驻服务：

```bash
./scripts/service/uninstall.sh
```

`launchd` 只保证用户会话中的进程拉起，不会代替 macOS 唤醒机制。电脑睡眠时的定时任务可能错过；盘前任务的误触发容忍为 60 秒，盘后为 300 秒。

### 4.3 前端开发服务（可选）

必须先让后端运行在 8765 端口，再执行：

```bash
./scripts/web/start.sh
./scripts/web/status.sh
./scripts/web/stop.sh
```

开发地址是 `http://127.0.0.1:5173/`，`/api` 会代理到 8765 端口。

## 5. 每日运行规则

常驻服务在 `Asia/Shanghai` 时区中调度：

| 时间 | 任务 | 行为 |
| --- | --- | --- |
| 07:50 | 盘前 | 按上交所日历获取截至前一交易会话的双源历史，验证连续 60 个合格会话并计算本地特征；DeepSeek 仅判断方向、置信度和理由，再由本地策略确定价格与仓位，通过组合风控后冻结模拟订单 |
| 18:15 | 盘后 | 用当日双源行情做保守成交判定，入账模拟成交并记录日终净值 |

盘前计划的最晚决策时间是 08:45。超时、上交所非交易日、时区缺失或数据不可用都会停止交易并返回 `HOLD`，不会产生模拟订单。A 股已使用 `exchange-calendars` 的上交所日历；港交所和临时休市仍需进一步补齐。

同日 18:00 前手工触发盘后任务会返回 `POSTMARKET_CUTOFF_NOT_REACHED`，并保留待处理订单与现金预留，不会提前结算或释放资金。

每个账户、每个交易日、每个标的在数据库中最多只能有一笔模拟订单。同一天可以操作多个不同标的。

### 5.1 `AGGRESSIVE_V2` 盘前规则

盘前会为每个人民币标的请求截至证据日的 180 个自然日区间，并逐日进行 Yahoo Finance / Tiingo 双源配对。历史特征统一使用复权 OHLCV；只有双源均存在，标的/日期/币种一致，收盘价偏差不超过 100 bps，开盘/最高/最低价偏差不超过 200 bps，且成交量对称偏差不超过 40% 的会话才会进入合格历史。合格成交量取两源较小值，避免放大流动性仓位。当日模拟成交仍使用原始价格。系统要求这些数据正好覆盖按上交所日历连续的最后 60 个交易会话；日期缺失、最后一根错位或仍出现超过 25% 的疑似未复权跳空时，该标的不会进入 AI 决策。

系统使用最后 60 根合格日线在本地计算 `SMA20`、`SMA60`、`ATR20`、20 日均量、20 日支撑和阻力，再把最新行情与这些聚合特征交给 DeepSeek。AI 的输出范围严格限制为方向、置信度和理由，订单参数由本地代码决定：

- 置信度低于 60% 时转为 `HOLD`。
- AI 给出买入后，本地仍要求 `收盘价 > SMA20 ≥ SMA60`；下降趋势中的买入会被拦截。
- 买入限价取 `min(收盘价 + 0.25 × ATR20, 收盘价 × 1.02)`；卖出限价取 `max(收盘价 - 0.25 × ATR20, 收盘价 × 0.98)`，并按标的最小价位保守取整。
- 买入止损距离取 `max(2 × ATR20, 收盘价 × 8%)`，既用于风险仓位计算，也会随实际成交买单成为持仓的活动止损参考。以后盘前若收盘价跌破该价，本地规则会在 AI 漏报或 DeepSeek 不可用时独立生成卖出意图。系统目前没有盘中独立止损订单，止损只在下一次盘前判断，隔夜跳空可能越过止损价。
- 单笔风险预算为 `账户净值 × 1.5% × AI 置信度`；同时受股票 30% / ETF 或 LOF 50% 仓位上限、20 日均量 5%、剩余现金、0.5% 费用缓冲和每日新增风险 50% 约束。
- 数量必须向下取整到整手，不足一手时转为 `HOLD`。卖出意图只退出当前可卖数量，不会超过 `sellable_quantity`；无可卖持仓时转为 `HOLD`。

多个买入信号按置信度从高到低共享剩余现金与每日新增风险额度。随后整份计划还要通过市场暴露、行业暴露、换手率和回撤等确定性组合风控；任一非 `HOLD` 提案失败时，不会冻结该计划中的模拟订单。

需要手动重放时应使用脚本：

```bash
./scripts/jobs/run_premarket.sh --date YYYY-MM-DD
./scripts/jobs/run_postmarket.sh --date YYYY-MM-DD
./scripts/jobs/run_backtest.sh --start YYYY-MM-DD --end YYYY-MM-DD
```

只读验证 Tiingo Token 和双源行情时使用以下脚本。它们不会调用 DeepSeek，也不会创建计划或订单：

```bash
./scripts/jobs/check_tiingo_token.sh
./scripts/jobs/run_data_smoke.sh --date YYYY-MM-DD
./scripts/jobs/run_strategy_smoke.sh --date YYYY-MM-DD
```

`run_data_smoke.sh` 验证指定交易日的单日盘后行情；`run_strategy_smoke.sh` 验证以指定交易日为决策日的 V2 连续 60 会话历史和特征。两者均为只读。

盘前计划按日期幂等：当天计划已冻结时，再次运行只会返回 `ALREADY_COMPLETED`，不会重新计算。不要在正式调度前随意对当天执行盘前脚本。

当前回测 CLI 尚未接入历史证据库，会返回 `NO_DATA` / `HISTORICAL_EVIDENCE_UNAVAILABLE`；它不代表已完成历史回测。

## 6. 飞书通知

调度器会在盘前和盘后工作流返回后，调用 `lark-cli` 发送 Markdown 消息。盘前消息包含逐标的买入、卖出或持有方向、当前/目标数量、目标权重、限价、策略版本、本地审核状态和理由；盘后消息包含账户净值、当日盈亏、累计盈亏和当前回撤。

- 未配置 `FEISHU_TARGET_ID` 时，工作流继续执行，只在日志中记录“跳过通知”。
- `lark-cli` 失败不会改写交易结果，只会记录错误。
- 当前使用确定性幂等键避免同一类日报重复发送，但调度器尚未把通知写入可重试的持久化 outbox。发送失败后需要人工核对日志。
- 自动新增自选标的尚未实现，因此 `WATCHLIST_ADDED` 类型目前没有生产触发入口。

## 7. 只读 API

仪表盘 API 只查询 SQLite，不建账户、不改持仓、不创建订单。

| 路径 | 用途 |
| --- | --- |
| `GET /api/health` | 运行状态，明确返回 `paper_trading_only=true` 和 `broker_connection=false` |
| `GET /api/account` | 初始资金、现金、市值、净值、盈亏、收益率和最大回撤 |
| `GET /api/watchlist` | 自选标的、最新价格、决策、数据状态和来源 |
| `GET /api/plans` | 最新交易计划摘要，包括策略审计理由和目标权重；旧记录缺失时使用兼容展示 |
| `GET /api/orders` | 模拟订单列表 |
| `GET /api/performance` | 日终净值、收益率和回撤序列 |
| `GET /api/data-health` | 数据库、行情源和已记录工作流的健康状态 |
| `GET /docs` | FastAPI 交互式 OpenAPI 文档 |

金额字段后缀 `_micros`，1 元 = 1,000,000 micros；比率字段后缀 `_bps`，100 bps = 1%。隐藏的兼容别名 `/api/overview` 和 `/api/trade-plans` 也可读，但不出现在 OpenAPI 列表中。

## 10. 常见故障

### 启动失败或 8765 端口无响应

1. 查看 `logs/app.log`；常驻模式同时查看 `logs/launchd.err.log`。
2. 确认没有同时运行手动服务和 `launchd` 服务。
3. 重新执行 `./scripts/setup.sh` 和 `./scripts/web/build.sh`，再用对应启动脚本重启。

### 每日计划一直是 `HOLD`

- 原因为 `TIINGO_TOKEN_MISSING`：配置 `TIINGO_API_KEY` 后重启服务。
- 原因为 `DUAL_SOURCE_DATA_UNAVAILABLE`：某一行情源缺数据、返回日期不符或请求失败；系统不会退化成单源下单。
- 双源日线未覆盖连续 60 个上交所交易会话、最后一根不等于证据日、OHLCV 超出容差或疑似含未复权除权跳空时，该标的不会进入 AI 决策。先用数据冒烟脚本检查两个来源及日期覆盖。
- AI 置信度低于 60%、本地风险预算买不起一个整手、没有可卖持仓、标的不可交易或缺少整手信息时，`AGGRESSIVE_V2` 会在本地转为 `HOLD`；查看计划理由可定位具体约束。
- 原因为 `PROPOSAL_OR_PORTFOLIO_DATA_INVALID`：检查 DeepSeek Key，并在日志中确认是鉴权、网络还是 JSON schema 失败。
- 原因为 `PREMARKET_CUTOFF_EXCEEDED`：已超过 08:45，当天不重做决策。
- 原因为 `RISK_REJECTED:*`：候选意图触发本地风控，应保留 `HOLD`。

### 盘后没有模拟成交

当日双源证据缺失、价格不在限价内、零成交量、涨跌停锁定、整手不合法或标的不可交易时，订单不会被伪造成交。数据不可用的待处理订单会标记为 `DATA_UNAVAILABLE` 并释放已预留现金。

### 飞书没有收到消息

1. 确认 `FEISHU_TARGET_TYPE` 与 `FEISHU_TARGET_ID` 匹配。
2. 确认在与常驻服务相同的 macOS 用户下，`lark-cli` 已登录且机器人有权向目标发送消息。
3. 查看 `logs/ai-trading.log` 中的“未配置收件人”或“飞书通知投递失败”。

## 11. 上线前仍需完成的验收

当前代码已提供模拟工作流和自动化测试，但不应将它描述为已完成正式策略验证：

- 日常盘前只会临时使用 60 个合格交易日做特征计算；完整历史行情、公司行动和历史交易规则尚未持久化并接入 CLI 回测流程。
- 尚未累计至少 20 个交易日的实际影子运行记录，也没有自动的“影子期通过”判定器。
- 港股交易日历、临时休市、停牌、逐标的涨跌停和完整公司行动复权仍需要更完整的公开数据证据；A 股已接入上交所日历，并对疑似未复权跳空停止决策。
- 港股、场内基金、科创板和创业板的生产级规则与数据路径尚未激活。

激进风控和 30% 硬回撤限制不代表收益保证，也不能防止行情缺口、流动性、数据错误或模型失误导致的模拟亏损。

# Phone2Computer 技术方案

## V2 目标

V2 在 V1 浏览器上传的基础上增加华为手机客户端。目标设备为华为 P40 Pro、HarmonyOS 4.2，交付形式为可侧载 APK。

- 支持一次选择数百张照片和视频。
- 切换到其他 App、回到桌面或屏幕熄灭后继续上传。
- App 或网络中断后，重新启动时从 Mac 已确认的偏移继续。
- 手机与 Mac 仍然只通过同一 Wi-Fi 直连，文件不经过云端。

## V2 架构

- 配对：手机扫码进入 V1 页面，点击“在 App 中继续”，通过 `phone2computer://pair` 深链传入局域网地址和高熵令牌。
- 文件授权：使用系统 `ACTION_OPEN_DOCUMENT` 批量选择照片和视频，并持久保留只读 URI 权限。
- 后台传输：Android `dataSync` 前台服务显示常驻通知，传输期间持有 CPU WakeLock 和高性能 Wi-Fi Lock。
- 断点续传：沿用 tus 1.0。每次恢复先向 Mac 发起 HEAD，以 Mac 已落盘偏移为准。
- 任务持久化：使用带 CRC 校验的追加式二进制日志。每完成一个 8MB 分片追加一条状态，异常截断只丢弃损坏尾记录。
- 并发与重试：最多 6 个文件并行；网络超时、429 和 5xx 最多重试 5 次，使用上限 30 秒的指数退避。
- 进度刷新：网络写入按 64 KB 上报字节数，界面和通知栏最多每 250 ms 刷新一次，避免高并发刷新拖慢传输。
- 防休眠边界：前台服务、CPU 锁和 Wi-Fi 锁覆盖正常锁屏场景；用户强行停止 App、关闭 Wi-Fi或系统极端省电仍会暂停，之后可以断点恢复。

Android 官方将文件上传列为 `dataSync` 前台服务的适用场景，并要求 Android 14 及以上声明对应服务类型和权限：

- [前台服务类型](https://developer.android.com/develop/background-work/services/fgs/service-types)
- [Storage Access Framework](https://developer.android.com/guide/topics/providers/document-provider)
- [共享文档访问](https://developer.android.com/training/data-storage/shared/documents-files)

## V2 交付链路

`scripts/mobile/build.sh` 构建 APK，并复制到 `dist/Phone2Computer-v2-debug.apk`。同时把 APK 放入 Web 静态资源，重新构建 Mac 服务后，手机可从扫码页面直接下载，不需要 USB。

## 目标

在同一 Wi-Fi 内，将华为 P40 Pro（HarmonyOS 4.2）的照片和视频批量上传到 macOS。手机无需安装应用，文件不经过云端。

## 已确认范围

- 手机通过浏览器扫码连接 Mac。
- 支持一次选择数百个照片和视频。
- 支持分片上传、暂停、重试和断点续传。
- 不保留手机原目录结构，Mac 按传输批次归档。
- 手机保持上传页面前台，不要求锁屏后台传输。
- 第一版只实现手机到 Mac 的单向传输。

## 架构

- Mac 接收端：Go 单进程 HTTP 服务，负责认证、tus 上传、文件落盘和静态资源服务。
- 手机与 Mac 页面：React + TypeScript，同一套前端根据路径展示上传页或管理页。
- 配对：服务启动时生成一次性高熵令牌，管理页使用二维码传递。
- 存储：文件先进入临时目录，完成后原子移动到本次传输批次目录。

## 安全边界

- 服务仅用于可信局域网，不主动开放公网入口。
- 所有上传接口要求 Bearer Token。
- 管理接口只允许本机回环地址访问。
- 服务端清理文件名并阻止路径穿越。
- 同名文件自动编号，默认不覆盖已有文件。

# 自动剪辑视频工作流

本项目把视频剪辑拆成三个阶段：素材分析、剪辑计划生成、按计划导出成片。所有运行入口都在 `scripts/` 目录，运行日志统一写入 `logs/`。

## 目录约定

- `inputs/`：原始视频素材。
- `outputs/`：分析结果、剪辑计划和最终视频。
- `logs/`：脚本和 Python 工作流日志。
- `scripts/`：所有运行、调试、测试入口。
- `src/auto_video_editing/`：自动剪辑实现代码。
- `tests/`：核心剪辑决策测试。

## 使用步骤

1. 初始化环境并检查工具：

```bash
scripts/bootstrap.sh
```

2. 把原始视频放入 `inputs/`，如果只有一个视频，可以不传参数。

3. 如果 `inputs/` 里有多个视频，先合并为统一分析源：

```bash
scripts/run_prepare.sh
```

4. 分析素材，提取音频、检测静音并调用 Whisper 转文字：

```bash
scripts/run_analyze.sh inputs/demo.mp4
```

多视频场景使用：

```bash
scripts/run_analyze.sh outputs/analysis/source.mp4
```

5. 生成剪辑计划 JSON：

```bash
scripts/run_plan.sh inputs/demo.mp4
```

6. 按计划执行剪辑，导出 60 秒以内、9:16、带字幕的视频：

```bash
scripts/run_edit.sh
```

也可以一键执行完整流程：

```bash
scripts/run_all.sh inputs/demo.mp4
```

## 输出文件

- `outputs/analysis/audio.wav`：从原视频提取的单声道音频。
- `outputs/analysis/transcript.json`：Whisper 字幕时间轴。
- `outputs/analysis/silences.json`：ffmpeg 检测出的静音片段。
- `outputs/plan.json`：可审查的剪辑计划。
- `outputs/work/subtitles.ass`：导出阶段生成的字幕文件。
- `outputs/work/captions/`：缺少 `subtitles` 滤镜时生成的透明字幕贴图。
- `outputs/final.mp4`：最终成片。

如果本机 ffmpeg 编译时包含 `subtitles` 滤镜，导出阶段会把 ASS 字幕烧录进画面。如果缺少该滤镜，工作流会用 Pillow 生成大号透明字幕贴图，再通过 ffmpeg `overlay` 滤镜按时间轴烧录进画面。

## 剪辑规则

- 删除 0.8 秒以上的静音间隔：剪辑计划只保留有字幕文本的片段，拼接时自然压缩字幕片段之间的长停顿。
- 删除低信息密度片段：对口癖、填充词、过短表达和低关键词文本降权。
- 控制总时长：优先选择信息密度更高的片段，总时长不超过 60 秒；最后一个片段必要时会被裁剪到剩余时长。
- 统一竖屏输出：导出时用 ffmpeg 缩放裁切为 1080x1920。
- 字幕导出：始终输出硬字幕；本地 ffmpeg 不支持 `subtitles` 滤镜时，自动改用 PNG 贴图叠加。
